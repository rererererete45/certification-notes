# 정답 및 간단 해설

> 아래 실행 결과는 각 `.c` 파일의 현재 코드 기준입니다.
> `printf()` 서식 문자열에 공백이 없으면 출력값도 붙어서 표시됩니다.

---

## 문제 02-01

### 실행 결과

```text
1012a
```

### 간단 해설

* `%d`는 10진수, `%o`는 8진수, `%x`는 16진수 소문자 출력이다.
* 10진수 `10`은 8진수로 `12`, 16진수로 `a`다.
* 서식 문자열에 공백이 없으므로 `1012a`로 붙어서 출력된다.

---

## 문제 02-02

### 실행 결과

```text
A
B	C
```

### 간단 해설

* `\n`은 줄바꿈이다.
* `\t`는 탭 간격이다.
* 탭의 실제 폭은 실행 환경에 따라 다르게 보일 수 있다.

## 문제 02-03

### 실행 결과

```text
65
```

### 간단 해설

* `a++`는 후위 증가 연산자다.
* `b`에는 증가 전 값인 `5`가 먼저 대입된다.
* 이후 `a`가 `6`이 되어 `a=6`, `b=5`가 된다.

---

## 문제 02-04

### 실행 결과

```text
1
1
```

### 간단 해설

* `a > 2 && b < 10`은 둘 다 참이므로 `1`이다.
* `a == 4 || b == 5`는 하나가 참이므로 `1`이다.
* C언어에서 논리식 결과는 참이면 `1`, 거짓이면 `0`이다.

---

## 문제 02-05

### 실행 결과

```text
21513
```

### 간단 해설

* `10 & 7`은 `2`다.
* `10 | 7`은 `15`다.
* `10 ^ 7`은 `13`이다.
* 공백이 없으므로 `21513`으로 출력된다.

---

## 문제 02-06

### 실행 결과

```text
205
```

### 간단 해설

* `10 << 1`은 비트를 왼쪽으로 한 칸 이동해 `20`이 된다.
* `10 >> 1`은 비트를 오른쪽으로 한 칸 이동해 `5`가 된다.
* 공백이 없으므로 `205`로 출력된다.

---

## 문제 02-07

### 실행 결과

```text
7
```

### 간단 해설

* `a > b`가 참이므로 삼항 연산자는 `a - b`를 선택한다.
* 따라서 `10 - 3`인 `7`이 저장된다.

---

## 문제 02-08

### 실행 결과

```text
16
```

### 간단 해설

* `a += 3`은 `a = a + 3`과 같다.
* `a *= 2`는 `a = a * 2`와 같다.
* `5 → 8 → 16` 순서로 계산된다.

---

## 문제 02-09

### 실행 결과

```text
66B
```

### 간단 해설

* 문자 `'A'`의 ASCII 값은 `65`다.
* `c + 1`은 `66`이다.
* `%d`는 숫자 `66`을 출력하고, `%c`는 ASCII 문자 `B`를 출력한다.

---

## 문제 02-10

### 실행 결과

```text
816
```

### 간단 해설

* `010`은 8진수 표기이며, 10진수로 `8`이다.
* `0x10`은 16진수 표기이며, 10진수로 `16`이다.
* 공백이 없으므로 `816`으로 출력된다.

---

## 문제 02-11

### 실행 결과

```text
4515
```

### 간단 해설

* `a++`는 현재 값 `3`을 먼저 사용한 뒤 `a`를 `4`로 증가시킨다.
* `++b`는 먼저 `b`를 `5`로 증가시킨 뒤 연산에 사용한다.
* 따라서 `result`는 `3 × 5 = 15`다.
* 최종 값은 `a=4`, `b=5`, `result=15`다.

---

## 문제 02-12

### 실행 결과

```text
10
```

### 간단 해설

* `x > 3`이 참이므로 `x << 1`이 실행된다.
* `5 << 1`은 `10`이다.

## 문제 02-13

### 실행 결과

```text
F
70
```

### 간단 해설

* `'A'`의 ASCII 값은 `65`다.
* `65 + 5`는 `70`이며, ASCII 문자로는 `'F'`다.
* `putchar(c)`는 문자 `F`를 출력한다.
* `printf("%d\n", c)`는 숫자 `70`을 출력한다.

---

## 문제 02-14

### 실행 결과

```text
5
```

### 간단 해설

* `10 / 3`의 몫은 `3`이다.
* `10 % 3`의 나머지는 `1`이다.
* `3 + (1 × 2)`이므로 결과는 `5`다.

---

## 문제 02-15

### 실행 결과

```text
33
```

### 간단 해설

* `a ^= b`는 `a = a ^ b`와 같다.
* `4 ^ 7`의 결과는 `3`이다.
* 이어서 `b &= a`는 `7 & 3`이므로 `3`이다.
* 최종적으로 `a=3`, `b=3`이 된다.
