<details>
<summary><strong>문제 01-01. 변수의 기억장소 유지 기간</strong></summary>

<br>

### 문제

> 다음 설명과 관계가 있는 C언어 변수의 종류를 쓰시오.
>
> “프로그램의 실행 시작 시점에 기억장소를 할당받아 실행이 끝날 때까지 지속적으로 기억장소를 유지한다.”

### 정답

**전역 변수(Global Variable) 또는 정적 변수(Static Variable)**

### 설명

C언어의 변수는 선언 위치와 저장 방식에 따라 기억장소를 유지하는 기간이 달라진다.

| 변수 종류 | 선언 위치                | 기억장소 유지 기간      |
| ----- | -------------------- | --------------- |
| 지역 변수 | 함수 또는 블록 내부          | 함수나 블록이 실행되는 동안 |
| 전역 변수 | 함수 외부                | 프로그램 시작부터 종료까지  |
| 정적 변수 | `static` 키워드를 사용해 선언 | 프로그램 시작부터 종료까지  |

#### 지역 변수

지역 변수는 함수 또는 블록 내부에서 선언한다.

함수가 호출되거나 블록이 실행될 때 기억장소가 할당되며, 해당 함수나 블록의 실행이 끝나면 기억장소에서 사라진다.

```c
void function(void)
{
    int number = 10;
}
```

#### 전역 변수

전역 변수는 모든 함수의 외부에서 선언한다.

프로그램 실행 시 기억장소의 데이터 영역에 할당되며, 프로그램이 종료될 때까지 유지된다.

```c
int global_number = 10;
```

#### 정적 변수

정적 변수는 `static` 키워드를 사용하여 선언한다.

함수 내부에 선언하더라도 기억장소는 프로그램이 종료될 때까지 유지되므로, 함수 호출이 끝난 뒤에도 이전 값이 보존된다.

```c
void function(void)
{
    static int count = 0;
    count++;
}
```

<details>
<summary><strong>문제 01-02. 올바르지 않은 변수명</strong></summary>

<br>

### 문제

> 다음 변수 선언 중 올바르지 않은 변수명을 모두 고르시오.

```text
㉠ LHJ_1
㉡ $TIME
㉢ 1_1
㉣ name and address
㉤ long
㉥ doublk
㉦ h_birth
㉧ __ok
㉨ start*key
```

### 정답

**㉡ `$TIME`, ㉢ `1_1`, ㉣ `name and address`, ㉤ `long`, ㉨ `start*key`**

### 식별자 작성 규칙

C언어의 식별자는 변수, 함수, 배열 등에 붙이는 이름을 의미한다.

식별자를 작성할 때는 다음 규칙을 지켜야 한다.

1. 영문자 또는 밑줄(`_`)로 시작해야 한다.
2. 두 번째 문자부터 영문자, 숫자, 밑줄을 사용할 수 있다.
3. 공백을 포함할 수 없다.
4. 밑줄을 제외한 특수문자를 사용할 수 없다.
5. C언어의 예약어를 사용할 수 없다.
6. 영문자의 대문자와 소문자를 구분한다.

### 변수명별 판단

| 번호 | 변수명                | 사용 가능 여부 | 판단 이유                           |
| -- | ------------------ | :------: | ------------------------------- |
| ㉠  | `LHJ_1`            |    가능    | 영문자로 시작하며 숫자와 밑줄 사용 가능          |
| ㉡  | `$TIME`            |    불가능   | 사용할 수 없는 특수문자 `$` 포함            |
| ㉢  | `1_1`              |    불가능   | 숫자로 시작                          |
| ㉣  | `name and address` |    불가능   | 변수명에 공백 포함                      |
| ㉤  | `long`             |    불가능   | C언어의 예약어                        |
| ㉥  | `doublk`           |    가능    | 예약어가 아닌 일반 식별자                  |
| ㉦  | `h_birth`          |    가능    | 영문자로 시작하며 밑줄 사용 가능              |
| ㉧  | `__ok`             |    주의    | 문법상 식별자 형태지만 표준 C에서는 구현체 예약 식별자 |
| ㉨  | `start*key`        |    불가능   | 사용할 수 없는 특수문자 `*` 포함            |

### 틀린 변수명의 수정 예시

```c
int time_value;
int value_1_1;
char name_and_address[100];
long long_value;
int start_key;
```

### 참고: `__ok` 사용 시 주의점

기초 문제에서는 밑줄로 시작할 수 있다는 규칙에 따라 `__ok`를 올바른 변수명으로 처리할 수 있다.

그러나 표준 C에서 밑줄 두 개(`__`)로 시작하는 식별자는 컴파일러와 표준 라이브러리 구현을 위해 예약되어 있다. 따라서 실제 프로그램에서는 다음과 같이 일반적인 이름을 사용하는 것이 안전하다.

```c
int ok;
int is_ok;
int ok_status;
```

## 핵심 정리

### 기억장소 유지 기간

```text
지역 변수 → 함수 또는 블록이 실행되는 동안 유지
전역 변수 → 프로그램 시작부터 종료까지 유지
정적 변수 → 프로그램 시작부터 종료까지 유지
```

### 변수명 작성 규칙

```text
가능: 영문자, 숫자, 밑줄(_)
시작 문자: 영문자 또는 밑줄(_)
불가능: 공백, 특수문자, 예약어, 숫자로 시작하는 이름
```
